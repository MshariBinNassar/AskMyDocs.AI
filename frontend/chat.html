What are the main constraints for the local storage implementation?

Sent 10:24 AM
smart_toy

Based on the provided documentation, the local storage implementation is subject to several key technical and security constraints:

  • Capacity Limit: The browser-side persistence is capped at 50MB per origin to ensure compatibility across older mobile Safari versions.
  • Encryption: All locally stored snippets must be encrypted using AES-256-GCM before writing to IndexedDB.
  • Latency: Read operations must complete within <100ms to avoid UI jank during document indexing.
  • Persistence: Sensitive user metadata cannot be cached permanently and must expire after the session token invalidates.
